## The Principle

Math is universal. A shader is just math. The execution substrate — GPU, CPU,
NPU, Android ARM core — is a hardware implementation detail. There is no
reason a shader can't be the math for any processor. That's silicon, not algebra.

Three primals each solve their portion:
- **barraCuda** solves the math (Layer 1)
- **coralReef** solves the compiler (Layers 2-3)
- **toadStool** solves the hardware (Layers 3-4)

Together they produce a stable, sovereign, pure Rust compute stack.

| Milestone | Status |

| SPIR-V → SASS compilation pipeline | Done |
| `nak-ir-proc` unsafe → safe (array-field or bytemuck) | Remaining |

### coralReef Unsafe Evolution Path
The 2 `unsafe` blocks in `nak-ir-proc` use `slice::from_raw_parts` on
`#[repr(C)]` structs with compile-time contiguity proofs via `offset_of!`.

Safe alternatives:
1. **Array-field pattern**: Store `Src`/`Dst` fields as `[Src; N]` with named
   accessors generated by the proc macro
2. **bytemuck cast**: If fields implement `Pod`, use `bytemuck::cast_ref` /
   `cast_mut` for safe reinterpretation
3. **Individual accessor**: Generate per-field methods returning `&T` instead
   of a contiguous slice (changes API but is fully safe)

### coralReef C Dependency Status
Zero application C deps. `libc` only via tokio/jsonrpsee (kernel ABI).
No `ash`, no `libloading`, no GPU FFI — coralReef is a pure Rust compiler.

## Validation Invariant

The physics never changes. The math is validated at every layer evolution.
Only the infrastructure evolves.

```
Springs find gaps → barraCuda fixes math → coralReef fixes compiler →
toadStool fixes hardware → Springs validate again
```

Each primal ingests, catches up, develops, and hands back. The cycle
accelerates: hotSpring precision shaders benefit wetSpring bio shaders,
neuralSpring ML shaders inform all springs' NPU paths, coralReef compiler
improvements benefit every shader barraCuda has ever written.
